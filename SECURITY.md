## üö® Reporting Security Vulnerabilities

**Do not report security vulnerabilities through public GitHub issues.**

### Responsible Disclosure Process

1. **Email**: security@smarthaus.group (subject: "SECURITY: [Brief Description]")
2. **Response**: Acknowledgment within 24 hours with case tracking number
3. **Investigation**: Regular updates during our investigation
4. **Resolution**: Security advisory and patch release
5. **Credit**: Recognition in security advisories (if desired)

### What to Include

- Detailed technical description
- Steps to reproduce (without exploits in public)
- Impact assessment (confidentiality, integrity, availability)
- Affected components and versions
- Suggested mitigation approaches
