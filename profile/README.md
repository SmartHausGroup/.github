# SMARTHAUS

**AI you can prove. Not AI you have to trust.**

SMARTHAUS builds deterministic governance systems for AI work where a model
output alone is not enough. Our focus is proof-bound software: systems that can
show what happened, why it was allowed, what evidence supported it, and what
would have stopped it.

## What We Build

SMARTHAUS work is organized around governed AI execution, deterministic runtime
behavior, proof-backed development, and controlled package supply chains for AI
capabilities.

Current public product descriptions are intentionally concise. Release status,
distribution readiness, and customer commitments are controlled by the owning
product repositories and approved release evidence.

## For Developers

This organization profile repository is a public front door. Product code,
release artifacts, detailed standards, and internal operating procedures live in
their owning repositories and systems.

When in doubt:

- use product repositories for code and releases;
- use published product documentation for installation and usage;
- use private reporting channels for vulnerability details;
- avoid putting secrets or private operational context in public issues.
